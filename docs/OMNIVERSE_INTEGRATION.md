# OASIS Omniverse Integration (OpenUSD)

## 1. Overview
As part of the Hyper-Scale architectural upgrade, the OASIS ecosystem connects the real-time interaction capabilities of the Unity ECS engine (OASIS_PRIME) with the photorealistic simulation capabilities of NVIDIA Omniverse.

## 2. Core Architecture: OpenUSD
The integration is entirely facilitated by **Universal Scene Description (OpenUSD)**.
Rather than using proprietary formats, Unity acts as a client connected to the Omniverse Nucleus server.

### 2.1 Unity Setup
Since we are targeting modern Unity versions, we use the **Unity Native USD packages**:
- `com.unity.importer.usd`
- `com.unity.exporter.usd`
- `com.unity.usd.core`

### 2.2 Live Sync Workflow
1. **ECS to GameObjects (Bridging):** High-velocity ECS entities (like drones simulated by `MovementSystem.cs`) push their transform data to proxy GameObjects.
2. **USD Export:** The `com.unity.exporter.usd` package continuously serializes these GameObjects, their materials, and physics colliders into USD layers.
3. **Omniverse Nucleus:** The USD layers are synchronized over the local network to an Omniverse Nucleus instance.
4. **USD Composer / Isaac Sim:** The Omniverse applications ingest the Live Sync, allowing robotics engineers (from Boston Dynamics / Tesla teams) to train models in Isaac Sim using data generated by the OASIS ecosystem.
